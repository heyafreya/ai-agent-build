K8s Health Monitor

An AI agent that monitors Kubernetes clusters

Project Summary

Why Architecture Components Alert Threshold Learnings Usage

Why

K8s dashboards require manual inspection. This project delivers an automated plain-English health summary via an LLM, with a deterministic severity score underneath that never lies.

Architecture

Agent runs either from CLI or via web server. The ReAct loop uses a Language Model to decide which tools to call: GET_POD s. return a pod table, GET_LOGS retrieves recent entries, and DESCRIBE_POD returns details. Tools are executed by a Python controller.

CLI → agent.py (ReAct loop) → tools: get_pods, get_logs, describe
                            → (in parallel) alerts.py (deterministic scores)
                            → combined output

Alert Threshold

Uses pure functions solely based on PodInfo: status names (CrashLoop+ImagePull → Critical, Pending → Warning), restart counts (>=5 Critical, >=3 Warning), and ready container ratios (0 ready → Critical, partial → Warning). Output severity scores are aggregated for reporting.

Learnings & Challenges

LLMs fabricate data without guardrails. Without an explicit pod list in the system prompt, the model invents names like frontend-abc123 that don't exist. Adding EXISTING PODS to the prompt fixed this.

Random data needs caching. random.sample() in the composite scenario generated a different pod set on every call. The LLM would describe pods from one call that vanished in the next. A cache dict ensures consistency across /health, /pods, and /analyze.

Focus mode needs filtering at every layer. Selecting a pod isn't just an API parameter — charts, tables, and LLM context all need to respect the focus. The severity chart initially showed all 6 pods even when one was selected.

Mock data must cover success cases too. describe_pod() only had data for bad pods. Healthy pods returned empty descriptions, so the LLM invented CrashLoopBackOff. Added detailed mock output for all healthy pods.

Regex can't parse LLM markdown. The model outputs ##, ###, - **bold**: — chained regex replacements broke on nested structures. Switched to marked.js.

JS regex order matters. Replacing ## before ### turned ### Issues Found into ##Issues Found. Reversed the order.
